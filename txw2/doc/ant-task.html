TXWC Ant Task

TXW comes with a built-in Ant task to make the Ant integration easier. First you have to declare the task in the build task like this: 

<taskdef name="txwc" classname="com.sun.tools.txw2.TxwTask">
  <classpath>
    <fileset dir="path/to/txw/installation" includes="**/*.jar" />
  </classpath>
</taskdef>

Then use the task like this: 

<target name="...">
  ...
  <txwc schema="src/foo.rng"
    package="org.acme.foo"
    destdir="build/src"
    methodChaining="true" />
</target>

Synopsis

Many of the attributes have the equivalent command line options.

Attribute Description Value Required?
methodChaining Equivalent of the -h option. "true" or "false". Defaults to true. No
package The equivalent of the -p option. A package name, such as "org.acme.foo" Yes
schema The schema file to be compiled. A file. relative to the base directory of the build. Yes
syntax The format of the schema. Equivalent of the -x/-c option. "xml" or "compact" No. If missing, the format is guessed from the file extension.
destdir Equivalent of the -o option. A directory. relative to the base directory of the build. Yes

At this point, the TXWC task defines no nested element.
