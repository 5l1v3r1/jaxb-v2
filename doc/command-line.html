TXWC Command Line Options

To show the summary of the available command-line options, simply execute TXWC without any parameter, as follows: 

java -jar txwc2.jar

The -c/-x option

TXW supports both the XML syntax and the compact syntax of RELAX NG. By default, TXW looks at the file extension to decide the syntax. If it's ".rnc", it's assumed to be the compact syntax, and otherwise it's assumed to be the XML syntax.

Specifying the -x option causes TXW to assume the XML syntax, regardless of the file name. Similarly, specifying the -c option causes TXW to assume the compact syntax.

The -h option

Consider the example in the getting started document. By default, "leaf" methods like Address.name(String) and Address.email(String) return void. Therefore, you have to code like the follows to get the values written: 

a = addr.address();
a.name("Kohsuke Kawaguchi");
a.email("kohsuke.kawaguchi@sun.com");
a.email("kk@kohsuke.org");

When the -h option is turned on, those leaf methods will be generated to return the this object. Thus you can write the same code as: 

addr.address().name("Kohsuke Kawaguchi").email("kohsuke.kawaguchi@sun.com").email("kk@kohsuke.org");

This eliminates the need of a local variable, and it also makes your code more compact, at the expense of the reduced readability. The similar coding style is also seen in StringBuffer.

If you prefer to code like this, use the -h option.

The -o/-p option

The -o option takes an argument and controls where the generated source files go. The -p option specifies the Java package name into which the source code is generated. For example, if you specify -o build/src -p org.acme, then you'll see a lot of files generated into ./build/src/org/acme.
